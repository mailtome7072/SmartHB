# ADR-008: 실행 시 PIN 인증 옵션화 — 기기별 선택적 PIN 게이트

- **상태**: Accepted
- **날짜**: 2026-06-02
- **결정자**: SmartHB 개발팀 (원장 요청 — post-Sprint 12)
- **관련**: ADR-001(SQLCipher), ADR-004(Keychain), ADR-007(6자리 PIN 전환)

## Context (배경)

앱 잠금 인증은 ADR-007로 6자리 숫자 PIN으로 전환됐다. 흐름: PIN 입력 → PBKDF2(600K) + salt → 32바이트 키 유도 → OS Keychain 저장 → 동일 키로 SQLCipher DB 복호화(프로덕션).

50대 단독 운영자(원장 1인)가 **매 실행마다 PIN 6자리 입력**하는 것이 부담이라는 피드백이 있다. 양 PC(교습소 Windows / 자택 macOS)를 시점 분리하여 단독 사용하는 운영 특성상, 실행 시 인증을 생략하고 싶다는 요구.

### 제약 사항

- **암호화 키는 OS Keychain에 PC별 저장**되어 있다(ADR-004). 즉 "키를 이미 보유한 PC"에서는 PIN 비교 없이도 키 확보가 가능하다.
- 프로덕션 DB는 SQLCipher 암호화 + 클라우드 동기화 폴더 위치. PIN을 없애도 **DB 자체는 암호화 유지**되어야 한다(평문화 불가).
- PRD §5.5는 "앱 실행 시 인증 의무"를 규정 — 본 결정은 이를 **기기별 선택적으로 완화**한다.
- 인증 토글 상태를 암호화 DB(app_settings)에 두면 **닭-달걀 문제**(키 없이 못 읽음).

---

## Decision (결정)

**"실행 시 PIN 인증 사용" 설정 토글을 신설한다. 기본 ON. C안(키체인 자동 스킵)을 채택한다.**

- **ON (기본)**: 현행대로 LockScreen에서 PIN 입력 → `verify_password` → 키 확보 → 잠금 해제.
- **OFF**: startup에서 **OS Keychain에 이미 저장된 유도 키를 그대로 로드**하여 잠금 해제하고, PIN 입력 단계를 스킵한다. `verify_password`의 **비교 단계만 생략** — 암호화/키 유도 로직은 변경 없음.
- **토글 저장 위치**: 암호화 DB가 아닌 `app_config_dir/config.json`(현 `cloud_folder_path` 보관처)에 `skip_pin_on_launch` 플래그로 저장. **PC별(per-device) 로컬** — 클라우드 동기화 대상 아님.
- **키체인 키 부재 시(새 PC, 최초)**: 스킵 불가 → 그 PC에서 **최초 1회 PIN 입력 필수**(키체인 시딩). 이후 스킵. 즉 "기기당 한 번"만 입력.
- **최초 설치**: PIN 설정 흐름은 그대로 유지(키 출처 확보).

## Consequences (영향) — 보안 트레이드오프

**완화되는 것**: OFF 설정 시 PIN 게이트가 사라져, 해당 PC에 물리적/계정 접근 가능한 자는 앱 데이터를 열람할 수 있다. → **데이터 보호를 OS 사용자 계정 로그인 + Keychain ACL에 위임**한다.

**유지되는 것**:
- DB는 여전히 SQLCipher 암호화 — DB 파일/클라우드 폴더가 유출돼도 키 없이 복호화 불가.
- 키는 OS Keychain에 PC별 보관(동기화 X) — "한 PC에서 OFF"가 "모든 PC 무인증"이 되지 않음.
- 하드코딩 시크릿/평문 DB 없음 — 프로젝트 보안 규칙 위반 없음.

**수용 근거**: 단독 운영자가 본인 소유 PC를 OS 계정으로 보호하며 사용하는 환경에서, 매 실행 PIN은 보안 이득 대비 UX 비용이 과도하다. OS 계정 잠금이 1차 방어선 역할을 충분히 한다.

## Alternatives Considered (기각된 대안)

| 대안 | 설명 | 기각 사유 |
|------|------|----------|
| **A. 하드코딩/고정 PIN 자동입력** | 코드/설정에 PIN을 박아 자동 입력 | 하드코딩 시크릿 = 암호화 무력화 + 프로젝트 규칙 위반. 키 노출 시 전체 무방비 |
| **B. 평문 DB + 무인증** | SQLCipher 제거, 인증 제거 | PII(원생 정보) 평문 노출 + 클라우드 폴더 유출 시 즉시 유출. 대규모 암호화 인프라 제거 필요 |
| **C. 키체인 자동 스킵 (채택)** | 키체인의 기존 유도키로 무입력 잠금해제, 토글은 기기별 config | 암호화/키 유도 무변경, 데이터 보호 유지, UX 부담만 제거 — 유일하게 보안 불변식 보존 |
| **D. 생체인증(Touch ID 등)** | OS 생체인증으로 PIN 대체 | 별도 의존성·플랫폼 분기 부담, 장기 과제로 이연(A83) |

## 구현 메모

- 토글을 **절대 DB에 저장하지 말 것**(닭-달걀).
- 키는 PC별 키체인 — OFF는 "키 있는 그 PC에서만" 적용.
- macOS 키체인 접근 시 OS 프롬프트("항상 허용") 발생 가능 — dev 빌드는 재컴파일마다 재프롬프트. 프로덕션 서명 빌드에서는 1회.
- `cipher` feature 무관하게 `auto_unlock_with_keychain` 은 `get_cached_or_load_key()` 로 salt(파일)+key(keyring) 로드를 동일하게 수행한다. cipher OFF(개발) 빌드에서는 로드한 키를 PRAGMA key 에 쓰지 않을 뿐(평문 DB), 키체인 접근과 "키 존재=이 PC 인증 키 보유" 판정은 양 빌드 공통이다. (당초 "stub/즉시성공" 기술은 부정확 — Sprint 14 A91 에서 정정). 키 부재 시 양 빌드 모두 `KeyNotFound` 반환 → LockScreen 폴백. 개발 모드(Tauri 미동작, 브라우저)에서는 프론트 래퍼가 먼저 reject.
- 복구 코드는 Sprint 12에서 제거됨 — PIN 분실 대비 안내를 설정 토글 근처에 표시("PIN을 잊으면 앱 데이터를 초기화해야 합니다").
