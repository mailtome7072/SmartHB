## 기준 1: Pretendard 폰트 (PRD §5.7) — ✅ 준수

- `src/app/globals.css`: Pretendard 400/600/700 @font-face 선언, fallback(Malgun Gothic / Apple SD Gothic Neo).
- 본문 `body` 18px(`--font-size-body`), 행간 1.5(`--line-height-base`) — **권장 충족**.
- 헤더 h1 32 / h2 28 / h3 24 / h4 22 / h5 20 / h6 18px — 24pt+ 하한 충족(h1~h3).
- `text-xs`(12px)·`text-sm`(14px)는 보조 정보(정렬순서, 메타, 부가 안내)에 한정 사용 — **본문 텍스트가 아니므로 정당**. 강제 축소 전역 규칙 없음.
- 예외: FullCalendar 시간 열(`globals.css:185` 12px)은 캘린더 격자 라벨로 밀집 UI 특성상 허용.

**조치**: 없음 (현행 유지).

## 기준 2: WCAG AA 명도 대비 (4.5:1) — △ 일부 미달 → **수정 완료(Critical)**

배경 `--background #F9F7F4` + 본문 `--foreground #1A1A1A`(약 16:1)는 안전. 회색 보조 텍스트에서 미달 발생.

| 색상 | 배경 대비 추정 | 판정 | 조치 |
|------|---------------|------|------|
| `text-gray-400` (#9CA3AF) | ~2.5:1 | **미달** | **→ `text-gray-600`(#4B5563) 일괄 치환 (17건/11파일)** |
| `text-gray-500` (#6B7280) | ~4.8:1 | 경계(일반 텍스트 AA 통과) | 유지 — 일괄 변경 시 디자인 영향 큼 |
| `--accent #2563EB` + 흰색 | 충분 | 통과 | — |
| `--danger #DC2626` + 흰색 | 충분 | 통과 | — |

**조치(완료)**: `text-gray-400` → `text-gray-600` 일괄 치환. 빈 상태 안내문·비활성 텍스트·아이콘 기본색이 진해져 가독성 향상(진하게 하는 방향이라 부작용 없음).

**이연(Sprint 16)**: `text-gray-500`(67건)의 개별 맥락 검토 — 작은 글씨에 쓰인 경우만 선별 상향. 고대비 모드 토큰 재설계.

## 기준 3: 44×44px 최소 클릭 영역 — △ 주요화면 준수

- **주요 액션(저장/실행/네비/입력)은 `h-11`(44px)/`h-12` 직접 지정으로 준수** — settings/students/diagnosis 등 점검 화면 전반.
- `src/components/ui/button.tsx`(shadcn Button)의 작은 기본 크기(h-8 등)는 **실사용 0건**(import 1, `<Button>` 사용 0)이므로 접근성 영향 없음 — Explore 초기 보고의 과대평가 정정.
- **미달(밀집 UI)** → Sprint 16 이연:
  - 출결표 셀(`AttendanceGrid.tsx`) — 행 높이 명시 제어 부재. 44px 강제 시 테이블 레이아웃 대폭 변경 필요(재검증 리스크).
  - 공지문 편집 아이콘 버튼(`notices/page.tsx` h-9/w-9 = 36px) — 편집 툴바 밀집.
  - 캘린더 셀 — FullCalendar 격자 구조 의존.

**조치**: 이번 수정 없음(주요화면 이미 준수). 밀집 UI는 레이아웃 재설계가 필요해 Sprint 16에서 별도 처리.

## 기준 4: 키보드 단축키 7종 (PRD §5.7) — △

| 단축키 | 기능 | 감사 전 | 이번 조치 |
|--------|------|---------|----------|
| **Ctrl/⌘+F** | 글로벌 검색 포커스 | 미구현(V19서 제거) | ✅ **신규 구현** (`GlobalShortcuts`) |
| **Ctrl/⌘+N** | 신규 원생 등록 | 미구현 | ✅ **신규 구현** (`/students/new` 이동) |
| Ctrl/⌘+Z | Undo | 출결표 국소 동작 | 유지 (현행) |
| ESC | 다이얼로그 닫기 | Radix Dialog/AlertDialog 기본 동작 | 유지 (현행) |
| Ctrl/⌘+P | 인쇄 | WebView 기본 인쇄 | 유지 (현행) |
| F1 | 도움말 | 미구현 | **Sprint 16 이연** (도움말 화면 미설계) |
| Ctrl/⌘+S | 저장 | 미구현 | **Sprint 16 이연** (화면별 저장 맥락 정의 필요) |

**조치(완료)**: `src/components/layout/GlobalShortcuts.tsx` 신규 — AppShell에 1회 마운트(`/lock`·`/setup` 제외). Ctrl+F(WebView 기본 찾기 억제 후 검색바 포커스)·Ctrl+N 처리.

**이연 사유**: F1(도움말)은 대상 화면이 없고, Ctrl+S(저장)는 화면마다 저장 대상·동작이 달라 전역 단일 처리가 부작용을 유발 → 화면별 맥락 설계 후 Sprint 16. (50대 1인 사용자 특성상 단축키 과다 도입은 학습 부담)
