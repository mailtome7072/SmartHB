# Sprint 20 — 실 DB 보정 절차 & 퇴교취소 재생성 워크플로우

> 대상: 교습소 원장(1인 운영). 코드 배포(인스톨러 설치) 후 원장이 직접 수행.
> 관련: T1(청구 규칙 전환) / T3·T4(청구 삭제) / ADR-010(삭제 가드 B안)

---

## A. 잘못 생성된 7월 청구/수납 데이터 보정 (T5)

### 배경
T1 이전(달력월 기준)에는 7월 교습기간(예: 7/2~7/29) 종료 이후 입교한 원생(입교 7/30)까지
7월 청구가 잘못 생성될 수 있었다. T1로 **앞으로는** 재발하지 않지만, **이미 생성된** 오류
청구는 남아 있으므로 T3/T4의 삭제 기능으로 정정한다.

### 절차

1. **사전 백업 (필수)**
   - 앱을 정상 종료한다(종료 시 자동 exit 백업 생성).
   - 추가 안전장치로, 클라우드 동기화 폴더의 `smarthb/app.db` 파일을 수동으로 1부 복사해 둔다
     (예: `app.db.bak-2026-07-19`). 보정에 문제가 생기면 이 사본으로 되돌린다.

2. **대상 식별**
   - 앱 실행 → 청구관리 → 문제의 월(예: 2026-07) 선택.
   - **입교일이 해당 월 교습기간 종료일(예: 7/29) 이후인 원생**의 청구가 대상이다.
   - 원생 상세(원생관리)에서 입교일을 확인해 교습기간 종료 이후 입교인지 대조한다.

3. **삭제 실행 (T3/T4 기능)**
   - 청구관리 그리드에서 대상 청구 행의 **[삭제]** 버튼 → 확인 다이얼로그에서 원생명·금액·상태
     확인 후 삭제.
   - `payments`는 CASCADE로 함께 삭제되므로 수납관리 목록에서도 자동으로 사라진다.
   - ⚠️ **대상 청구가 수납완료(is_paid=1) 상태면 삭제가 막힌다(ADR-010 B안).** 이 경우
     **① 수납관리에서 해당 청구의 수납을 해제(입금완료 해제) → ② 청구관리에서 삭제** 순서로 진행한다.

4. **검증**
   - 삭제 후 청구관리 7월 목록을 재조회 → 대상 원생이 제외되었는지 확인.
   - 수납관리 7월 목록에서도 사라졌는지 확인.
   - (선택) 감사 로그에 `bill-deleted` 이벤트가 기록되었는지 확인.

5. **롤백 계획**
   - 보정 결과가 예상과 다르면 앱을 종료하고 1단계에서 만든 `app.db.bak-*` 사본으로
     `smarthb/app.db`를 덮어써 복원한 뒤 재시도한다.

## B. 퇴교 취소 → 월별 데이터 재생성 워크플로우 (T5-b)

### 배경
교습기간 종료 이후 입교한 신규 원생을 이슈 회피 목적으로 **일시 퇴교** 처리해 둔 상태에서,
다음 달 교습기간을 만든 뒤 **퇴교를 취소**했을 때의 올바른 후속 절차 안내다.

### 핵심 (검증된 현재 동작)
- **퇴교 취소도, 교습기간 등록도 청구·수납·교습일을 자동 생성하지 않는다.** 각각 수동 실행이 필요하다.
- 출결 생성·청구 생성은 재실행이 안전하다(`INSERT OR IGNORE`) — 이미 만든 다른 원생 데이터는
  중복·훼손 없이, 새로 재원 상태가 된 원생 몫만 추가된다.
- 출결 생성은 해당 월 교습기간이 **등록 + 확정**되어야 동작한다.

### 순서 (순서 중요)
1. **다음 달(예: 8월) 교습기간 등록 + 확정** — 출결 생성의 전제.
2. **원생 퇴교 취소(reinstate)** — 반드시 생성보다 **먼저**. 퇴교 상태로 두면 출결/청구 생성
   대상에서 제외된다(단, 취소 후 재실행하면 채워지므로 영구 유실은 아님).
3. **출결 생성 재실행** → 해당 원생의 교습일(정규출결) 생성.
4. **청구 생성 재실행** → 해당 원생의 청구 생성. (교습기간 미등록 월은 T1에 의해 차단되므로
   1번을 먼저 해야 한다.)
5. **수납 기록** — 필요 시 수납관리에서 수동 입력.

### 주의
- 8월 청구는 T1 수정과 무관하게 정상이다(8월 날짜는 8월 교습기간에 정확히 포함).
- 7월 교습일은 원래 0건이다(입교 7/30 > 7월 교습기간 종료 7/29 → 생성 대상 날짜 없음).
- 7월 과청구(있다면)는 위 A절 절차로 정정한다.
