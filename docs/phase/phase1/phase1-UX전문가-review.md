## 1. 초기 설정 마법사 (§4.0) — 9단계 적정성

### 분석
- PRD 원문: 9단계 (환영 → 운영시간 → 학교코드 → 표준교습비 → 결제수단 → 백업폴더 → 가져오기 → 샘플원생 → 완료)
- 50대 사용자 + 첫 실행: 9단계는 심리적 부담 유발 가능
- 각 단계 소요 시간 추정: 1~2분 × 9 = 9~18분 (PRD 원문 "약 10~15분")

### 권고사항

| 등급 | 항목 | 설명 |
|------|------|------|
| **중요** | 단계 수 축소 검토 | 5~6단계로 통합 권장: (1) 환영 (2) 데이터 저장 위치 (3) 운영시간+표준교습비 (4) 학교코드+결제수단 (5) 샘플 원생 (6) 완료. 가져오기는 설정 메뉴로 이관 |
| **보수적 채택** | PRD 9단계 유지 | PRD §4.0 원문을 존중하여 9단계 유지. 대신 아래 UX 보완 필수 적용 |
| **필수** | 진행률 표시 | 상단에 "1/9", "2/9" 스텝 인디케이터 + 진행률 바. 현재 위치와 남은 단계를 명확히 시각화 |
| **필수** | 건너뛰기 접근성 | "건너뛰기" 버튼 크기 44x44px 이상, "다음" 버튼과 동등한 시각적 위치. "나중에 설정 가능합니다" 안내 텍스트 |
| **필수** | 각 단계 독립 저장 | PRD AC-4.0-2 준수. 중간에 앱을 닫아도 진행분 보존. 재진입 시 "이전에 3단계까지 완료하셨습니다" 안내 |
| 권고 | 단계별 소요 시간 표기 | 각 단계 상단에 "약 2분 소요" 표시 — 50대 사용자에게 예측 가능성 제공 |
| 권고 | 마법사 완료 후 튜토리얼 | 완료 화면에서 "주요 기능 둘러보기" 선택지 — 사이드바 메뉴별 간단 설명 툴팁 |

## 5. 접근성 기준선 (§5.7, Sprint 3 확립)

### 체크리스트 (모든 화면 공통)

| 항목 | 기준 | 검증 방법 |
|------|------|----------|
| 본문 폰트 | Pretendard 18pt (16pt 하한) | CSS 검사 |
| 헤더 폰트 | 24pt 이상 | CSS 검사 |
| 행간 | 1.5 | `line-height` 검사 |
| 명도 대비 | >= 4.5:1 (WCAG AA) | axe 또는 Lighthouse 접근성 감사 |
| 클릭 영역 | 최소 44x44px | 모든 버튼/체크박스/링크 패딩 검사 |
| 배경색 | 저자극 톤 (베이지/연그레이) | 디자인 토큰으로 통일 |
| 강조색 | 무채도 기반 | 색맹 시뮬레이터 확인 |
