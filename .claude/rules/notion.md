## Notion MCP 사용 원칙

- Notion 업데이트는 반드시 사용자 확인 후 실행 — 자동 실행 금지
- 업데이트 후 반드시 페이지 URL을 보고한다
- 아래 페이지 ID가 미입력 상태이면 사용자에게 입력을 요청한 후 진행한다

## 자동 업데이트 트리거

에이전트 완료 보고 시 아래 이벤트에 해당하면 Notion 업데이트 필요 여부를 안내한다:

| 이벤트 | 업데이트 대상 페이지 |
|--------|---------------------|
| DB 스키마 변경 포함 스프린트 완료 | 데이터 모델 |
| API 엔드포인트 추가/변경 | API 명세 |
| 새 기능 릴리즈 (deploy-prod 완료) | 기능 명세, 릴리즈 노트 |
| 아키텍처 변경 | 시스템 아키텍처 |

## Notion 페이지 ID 매핑

> **초기 설정 방법**: Notion 연동을 시작할 때 이 파일(`.claude/rules/notion.md`)을 직접 열어 아래 테이블의 `(입력 필요)` 값을 실제 페이지 ID로 교체하세요.
> 페이지 ID는 Notion 페이지 URL 마지막 부분입니다: `https://notion.so/{워크스페이스명}/{페이지ID}`
> `/setup-project` 슬래시 커맨드는 이 파일을 자동 치환하지 않으므로 수동으로 편집해야 합니다.

| 용도 | 페이지 ID |
|------|----------|
| 시스템 아키텍처 | (입력 필요) |
| 데이터 모델 | (입력 필요) |
| API 명세 | (입력 필요) |
| 기능 명세 | (입력 필요) |
| 릴리즈 노트 | (입력 필요) |
| 스프린트 현황 | (입력 필요) |
