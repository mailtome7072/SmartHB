# code-review

sprint-close agent의 코드 리뷰 단계 및 hotfix-close agent의 경량 코드 리뷰 단계에서 이 체크리스트를 사용합니다.

## 리뷰 코멘트 라벨 (Conventional Comments)

코드 리뷰 코멘트 작성 시 아래 라벨을 접두사로 사용하여 의도를 명확히 한다. (출처: conventionalcomments.org, Google/Palantir 채택)

| 라벨 | 의미 | 처리 |
|------|------|------|
| `issue:` | 버그·보안·기능 오류 | 필수 수정 |
| `suggestion:` | 더 나은 방법 제안 | 선택 적용 (non-blocking) |
| `question:` | 의도·맥락 명확화 요청 | 답변 후 판단 |
| `nitpick:` | 사소한 스타일·네이밍 | 선택 수정 |
| `praise:` | 잘한 점 명시 | 응답 불필요 |

## PR 크기 가이드라인

리뷰어의 인지 과부하를 막고 결함 감지율을 유지하기 위한 기준. (연구 기반: SmartBear, 2024)

- **Sprint PR**: 200-400줄 이내 권장 / 초과 시 기능 단위로 분리 검토
- **Hotfix PR**: 50줄 이내 (dev-process.md 섹션 2 기준 — 변경된 코드 diff 기준)
- 400줄 초과 PR은 리뷰 품질이 급격히 저하되므로 분리를 우선 권고한다.
- 400줄 초과 PR 발견 시 `sprint-review`는 사용자에게 분리 검토를 먼저 권고하고 진행 여부를 확인합니다.

## AI 생성 코드 리뷰 추가 체크

이 프로젝트는 AI 에이전트가 코드를 직접 작성한다. 일반 체크리스트 외에 아래 항목을 추가로 확인한다. (출처: Microsoft Engineering Blog, 2024)

- ⬜ 비즈니스 로직이 실제 요구사항(sprint 계획 문서)과 일치하는가
- ⬜ 의도치 않은 파일 변경이 없는가 (`git diff --stat`로 확인)
- ⬜ 하드코딩된 테스트 데이터·더미 값이 프로덕션 코드에 남아있지 않은가
- ⬜ AI가 추가한 의존성이 기존 스택과 충돌하지 않는가

## 보안

- ⬜ 하드코딩된 시크릿, API 키, 비밀번호 없음
- ⬜ SQL 인젝션 방지 (ORM 파라미터 바인딩 사용)
- ⬜ XSS 방지 (React 기본 이스케이프 사용, 인라인 HTML 주입 최소화)
- ⬜ 인증/인가 체크 누락 없음

## 성능

- ⬜ N+1 쿼리 없음 (SQLAlchemy relationship 로딩 전략 확인)
- ⬜ 불필요한 API 호출 없음
- ⬜ 리스트 응답에 페이지네이션 적용

## 코드 품질

- ⬜ TypeScript 타입 안전성 (any 사용 최소화)
- ⬜ 에러 핸들링 (FastAPI HTTPException, 프론트엔드 에러 바운더리)
- ⬜ 구조화 로깅 (JSON 형식, Request ID 포함)

## 테스트

- ⬜ 새 기능에 pytest 테스트 추가 여부
- ⬜ 기존 테스트 회귀 없음 (`pytest -v` 통과)

## 패턴 준수

- ⬜ 프로젝트 컨벤션에 맞는 파일/디렉토리 구조
- ⬜ API 클라이언트 추상화 레이어 사용

## 리뷰 등급 및 처리 기준

sprint-close agent와 hotfix-close agent는 발견된 이슈를 아래 등급으로 분류하여 처리한다.

| 등급 | 기준 예시 | 처리 방법 |
|------|-----------|-----------|
| **Critical** | 하드코딩된 시크릿, SQL 인젝션, 인증 우회 가능성 | 배포 차단 — 반드시 수정 후 재검토 |
| **High** | 인증/인가 누락, 심각한 N+1 쿼리, 주요 기능에 테스트 없음 | 사용자 확인 후 배포 여부 결정 |
| **Medium** | 타입 안전성 미흡, 페이지네이션 누락, 로깅 부실 | 검증 보고서(risk-register)에 기록, 배포 진행 |
| **Low** | 네이밍 컨벤션, 주석 누락 등 스타일 이슈 | 기록만 함 |

## Hotfix 경량 리뷰 적용 범위

`hotfix-close` 에이전트가 경량 코드 리뷰를 수행할 때 아래 항목만 적용합니다:

| 적용 | 섹션 |
|------|------|
| ✅ 전체 적용 | 보안 섹션 전체, AI 생성 코드 추가 체크, 리뷰 등급 기준 |
| ⚠️ 관련 항목만 | 성능, 코드 품질, 테스트, 패턴 준수 (변경 파일 직접 관련 항목만) |
| ❌ 생략 | PR 크기 가이드라인 (hotfix 기준 50줄이 이미 적용됨) |
