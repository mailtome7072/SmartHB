# simplify

Task 구현 완료 후 작성된 코드를 검토하고, 불필요한 복잡성을 제거합니다.
Karpathy 원칙 4 ("복잡성을 피하고, 단순하게 유지한다")의 자동 적용 단계입니다.

## 적용 시점

`/sprint-dev` 실행 중 **스프린트 내 모든 Task 구현 완료 후 1회** 자동 실행됩니다.
수동 호출도 가능합니다: `"방금 작성한 코드 simplify 해줘."`

## 검토 항목

### 1. 불필요한 추상화 제거
- 단 하나의 호출자만 있는 함수/클래스를 별도 분리했는가?
- 미래를 위한 "혹시 모르니" 코드가 있는가?
- 현재 필요하지 않은 인터페이스/추상 클래스가 추가되었는가?

### 2. 중복 로직 제거
- 동일한 로직이 2곳 이상에 존재하는가?
- 공통 처리 가능한 조건문이 분산되어 있는가?

### 3. 코드 평탄화
- 중첩 조건문을 early return으로 단순화할 수 있는가?
- 더 단순한 자료구조로 해결 가능한가?
- 변수명/함수명이 역할을 명확히 드러내는가?

### 4. 사용하지 않는 코드 제거
- import했지만 사용하지 않는 모듈
- 선언했지만 참조하지 않는 변수
- 주석 처리된 코드 블록 (TODO가 아닌 것)

## 실행 절차

1. 이번 Task에서 수정/생성한 파일 목록을 확인한다.
2. 각 파일을 위 4가지 항목으로 검토한다.
3. 수정이 필요한 항목은 즉시 수정한다.
4. 결과를 아래 형식으로 보고한다.

## 출력 형식

```
simplify — {파일명}
  수정: {수정 내용 한 줄 요약}   (수정 없으면 "변경 없음")
```

## 주의사항

- **기능 변경 금지** — 동작은 동일하게, 코드 구조만 단순화한다.
- **허용**: 변수명 개선, 중복 코드 추출, early return 적용, 미사용 코드 제거 (동작 결과 동일 보장)
- **불허**: 알고리즘 교체, 성능이 다른 구현으로 변경, 인터페이스 변경 → 이 경우 별도 Task로 분리
- 리팩토링 범위가 현재 Task를 초과하면 수정하지 않고 ROADMAP에 후속 Task로 기록한다.
- 테스트가 있는 경우 simplify 후 테스트가 여전히 통과하는지 확인한다.
