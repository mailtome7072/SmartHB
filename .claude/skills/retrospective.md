# retrospective

sprint-close agent의 10단계(Sprint 회고 작성)에서 이 스킬을 참조합니다.
`docs/sprint-retrospectives/sprint{n}.md` 생성 시 아래 형식과 원칙을 준수합니다.

## 시작 전: Prime Directive

회고를 시작하기 전, 다음 원칙을 전제로 한다. (출처: Norm Kerth / Google SRE Blameless Postmortem)

> "이 회고에 참여하는 모든 사람은 그 시점에 주어진 정보, 역량, 자원, 상황 하에서 최선을 다했다고 가정한다."

- 개인을 탓하지 않고 **시스템과 프로세스 개선**에 집중한다.
- 문제의 원인을 찾을 때 "누가" 아닌 "무엇이" "왜"에 초점을 맞춘다.

## 문서 형식

```markdown
# Sprint Retrospective sprint{n}

## 잘한 점
- (계획 대비 완료된 목표, 잘 동작한 구현, 반복하고 싶은 패턴)

## 개선할 점
- (코드 리뷰에서 발견된 이슈, 검증 실패 항목, 예상보다 오래 걸린 작업)

## 액션 아이템
- (다음 스프린트에서 반영할 구체적 개선 사항 — 각 항목은 실행 가능한 단위로 작성)
```

## 작성 원칙

1. **데이터 기반으로 작성한다**
   - 스프린트 계획(`docs/sprint/sprint{n}.md`)과 실제 커밋 이력(`git log sprint{n} --oneline`)을 비교하여 작성한다.
   - 코드 리뷰 결과(4단계)와 검증 결과(5단계)에서 구체적 근거를 가져온다.
   - 데이터가 없는 항목은 "해당 없음"으로 표시하고 추측으로 채우지 않는다.

2. **액션 아이템은 SMART 기준으로 작성한다**
   - SMART: Specific(구체적) / Measurable(측정 가능) / Achievable(달성 가능) / Relevant(관련성) / Time-bound(기한 명시)
   - "더 잘하자" 수준의 추상적 표현 금지. 다음 스프린트 계획에 그대로 옮길 수 있을 만큼 구체적으로 작성한다.
   - 예시:
     - ❌ `"성능 개선"`
     - ✅ `"DB 응답 시간 200ms → 100ms로 개선 (다음 스프린트 내)"`
     - ❌ `"N+1 쿼리 줄이기"`
     - ✅ `"UserList API의 N+1 쿼리를 SQLAlchemy joinedload로 교체 (sprint{n+1} 백엔드 작업 목록에 추가)"`
   - SMART 기준 미충족 항목은 재작성한다. 기술적·일정 제약으로 현 시점에 구체화가 불가능한 경우만 `(기준 미정 — 다음 계획 시 구체화)` 표시 후 기록한다.

3. **이전 회고의 액션 아이템 이행 여부를 확인한다**
   - 직전 회고(`docs/sprint-retrospectives/sprint{n-1}.md`)의 액션 아이템이 이번 스프린트에서 반영되었는지 검토한다.
   - 미이행 항목은 "개선할 점"에 재기록하거나 다음 액션 아이템으로 이월한다.

4. **sprint-planner가 다음 계획에 활용할 수 있게 작성한다**
   - 액션 아이템은 writing-plans 스킬의 "이전 회고 반영" 섹션에 그대로 연결된다.
   - 기술 부채, 리스크, 의존성 발견 사항은 명확하게 기술한다.

## 대안 회고 프레임워크

기본 포맷(잘한 점/개선할 점/액션 아이템)의 반복으로 회고 피로가 생길 경우 아래 프레임워크로 교체할 수 있다. 산출물 파일 구조는 동일하게 유지한다.

### Four Ls (감정 + 학습 균형형)
> 출처: Atlassian Team Playbook

| 섹션 | 질문 |
|------|------|
| **Liked** | 이번 스프린트에서 좋았던 점은? (감정 기반) |
| **Learned** | 새롭게 배우거나 발견한 것은? (데이터 기반) |
| **Lacked** | 부족했거나 누락된 것은? |
| **Longed For** | 있었다면 더 좋았을 것은? (미래 지향) |

### Start / Stop / Continue (행동 중심형)
> 출처: Scrum.org / Atlassian

| 섹션 | 질문 |
|------|------|
| **Start** | 앞으로 시작해야 할 새로운 관행은? |
| **Stop** | 성과 없거나 방해가 되는 것은? |
| **Continue** | 잘 작동하고 있어 유지해야 할 것은? |
