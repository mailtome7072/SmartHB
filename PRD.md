## 1. Executive Summary

### 1.1 프로젝트 개요
초등 및 예비중등 학생 대상 1:1 맞춤 수학 교습소(스마트해법수학 서현효자점)의 운영 전반을 단일 데스크톱 애플리케이션으로 관리한다. 원생 관리, 수업 스케줄, 출결/보강, 단원평가, 교습비 청구·수납, 카카오톡 공지문 이미지 생성, 대시보드를 통합 제공한다.

### 1.2 핵심 가치 제안
- 흩어진 운영 정보를 단일 도구로 통합
- 50대 사용자도 무리 없이 다룰 수 있는 UI/UX(큰 글자, 저자극 색상)
- 출결 → 보강 → 청구의 일관된 자동 데이터 흐름으로 입력 누락·계산 실수 방지
- Windows(교습소) ↔ Mac(자택) 시점 분리 사용 + OS 클라우드 동기화 폴더로 데이터 공유

### 1.3 대상 사용자
- 50대 교습소 원장 1인(단독 운영자, 단일 계정)

## 2. Objectives & Goals

### 2.1 비즈니스 목표
- 교습비 미납률 감소
- 월말 출결 데이터 입력 완료율 향상
- 월 행정 업무 처리 시간 단축

### 2.2 기술 목표
- Windows 10/11, macOS 12+ 동작
- 양 PC 간 데이터 일관성 보장 (시점 분리 사용 + OS 클라우드 동기화 폴더)
- 인터넷 단절 시에도 로컬 작업 가능 (연결 복구 시 OS 클라이언트가 자동 동기화)
- 50대 사용자 접근성: Pretendard 본문 18pt 권장(16pt 하한), 명도 대비 WCAG AA 이상

### 2.3 성공 기준(측정 가능)
- 미납 발생률 ≤ 5% (전년 대비 -50%)
- 출결 입력 완료율 ≥ 95% (월 단위)
- 월 행정 처리 시간 ≤ 4시간 (현재 대비 -50%)
- 사용자 학습 기간 ≤ 1주 (자가 학습으로 주요 기능 사용 가능)

## 3. Users & Use Cases

### 3.1 Target Users (Persona)

**원장 김OO (52세, 여성)**
- 역할: 교습소 단독 운영자(수업 + 행정 모두 담당)
- 디지털 친숙도: 보통(카카오톡·엑셀 기초·웹 검색)
- 환경: 교습소 Windows 데스크톱 + 자택 MacBook
- 페인포인트
  - 수업·행정 병행으로 출결 입력을 잊거나 미루는 경우 발생
  - 결석 → 보강 흐름을 수기 관리 → 미보강 누락
  - 매월 청구액 계산을 표 계산으로 처리 → 실수 위험
  - 카톡 공지문을 원생마다 수기 편집 → 시간 소요 큼

### 3.2 Primary Use Cases

**UC-1. 신규 원생 등록**
- 전제: 원장 로그인 상태
- 스텝: 원생 정보 입력 → 요일별 수업 시간 설정 → 입교일 지정 → 저장
- 결과: 원생 등록 + 주 수업시간 자동 계산 + 청구액 미리보기

**UC-2. 월말 학사 일정 수립**
- 전제: 다음 달 학사 미생성 상태
- 스텝: 학사 스케줄 화면 → 다음 달 교습기간 시작/종료 클릭 → 확정 → 보강데이·방학·단원평가 등 일정 배치
- 결과: 교습기간 + 학사일정 확정

**UC-3. 일일 출결 입력**
- 전제: 출결관리 데이터 생성 완료
- 스텝: 출결관리 메뉴 → 원생별 셀 클릭으로 출석/결석 토글
- 결과: 결석 시 보강필요시간 자동 누적

**UC-4. 결석 원생 보강 처리**
- 전제: 보강필요시간 보유 원생 존재
- 스텝: 출결표에서 보강 희망 일자 셀 클릭 → 충당할 결석 다중 선택 (소멸기한 임박 순)
- 결과: 보강 데이터 생성 + 선택된 결석들이 "보강완료"로 전이 (일 단위 매칭, 보강 1건 ↔ 결석 N건 가능)

**UC-5. 월 교습비 청구 및 카톡 공지문 발송**
- 전제: 청구월의 출결 마감 완료
- 스텝: 교습비 청구 메뉴 → 청구 생성 → (필요 시 개별 금액 조정) → 공지문 생성 메뉴 → 배경서식 선택 → 발송용 이미지 일괄 생성
- 결과: PNG 이미지 일괄 저장 → 원장이 카톡으로 수동 발송

**UC-6. 단원평가 점수 입력**
- 전제: 학사일정에 단원평가 등록됨
- 스텝: 단원평가 메뉴 → 해당 평가 선택 → 원생별 1차 점수 입력 → 오답 재시험 후 2차 점수 입력
- 결과: 평가 기록 저장 + 학년/원생별 추이 갱신

## 4. Core Features

### 4.0 초기 설정 마법사 (Setup Wizard)

**Feature 4.0.1 첫 실행 마법사**

앱 첫 실행 시(또는 데이터가 비어 있는 상태에서 시작 시) 단계별 마법사가 자동 진입한다. 50대 사용자가 빈 화면 앞에서 막막함을 느끼지 않도록 운영에 필요한 최소 데이터를 순서대로 가이드한다.

**마법사 단계**
1. 환영 화면 (앱 개요, 예상 소요 시간 안내 약 10~15분)
2. 교습소 운영 시간 설정 (요일별 시작·종료, §4.12.2 참조)
3. 학교 코드 추가 (최소 3개 권장, 추후 추가 가능)
4. 표준 교습비 매핑 (주 단위 수업시간 ↔ 금액, §4.3 참조)
5. 결제수단 활성화 확인 (기본 6종 중 사용 항목 선택)
6. 백업 폴더 경로 지정 (클라우드 동기화 폴더, §5.3 참조)
7. 원생 데이터 가져오기 (선택, §4.13 참조)
8. 샘플 원생 1명 등록 튜토리얼 (선택, 등록 흐름 체험용)
9. 완료 화면

**Feature 4.0.2 마법사 건너뛰기 및 재실행**
- 각 단계는 "건너뛰기" 가능, 추후 설정 메뉴에서 개별 설정
- 마법사 미완료 시 다음 실행에서 이어서 진행
- 설정 메뉴에 "초기 설정 마법사 다시 실행" 옵션 제공

**AC**
- AC-4.0-1: 첫 실행 또는 핵심 설정(운영 시간/표준 교습비/결제수단) 미설정 상태면 마법사 자동 진입
- AC-4.0-2: 마법사의 각 단계는 독립적 저장 (도중 종료해도 진행분 보존)
- AC-4.0-3: 마법사 완료 후 대시보드로 자동 이동

### 4.1 원생 관리

**Feature 4.1.1 원생 정보 등록/수정**

원생 정보는 **기본 정보**와 **요일별 수업 스케줄**로 구분된다.

**(가) 원생 기본 정보 (원생당 1건)**
- 일련번호(수강생 대장 법정번호, 수동 입력 또는 자동 채번)
- 이름
- **성별 (남/여)**
- 학교급(초등/중등)
- 학년(초1~6 / 중1~3)
- 학교명(학교 코드 테이블 참조)
- 입교일, 퇴교일
- 전화번호: 원생, 모, 부

**(나) 요일별 수업 스케줄 (한 원생당 요일별 1건)**
- 요일 (월~금 중 복수 선택 가능, 한 요일에 1건만 허용)
- 수업 시작 시간 (해당 요일의 교습소 운영 시간 내 1시간 단위)
- 1회 수업 시간 (해당 요일의 운영 종료 시간에 따라 최대값 자동 산출)
- 등록 후 변경 사항은 변경 이력에 누적 (§4.2.2 참조)

**수락 기준 (AC)**
- AC-4.1.1-1: 수업 요일은 1개 이상 선택해야 저장 가능
- AC-4.1.1-2: 수업 시작 시간 + 1회 수업 시간 ≤ 해당 요일의 교습소 운영 종료 시간 (§4.12 설정 참조)
- AC-4.1.1-3: 동일한 일련번호 중복 등록 불가
- AC-4.1.1-4: 퇴교일은 입교일 이후 일자만 허용
- AC-4.1.1-5: 1회 수업 시간 선택 UI는 시작 시간에 따라 가능한 옵션만 활성화

**Feature 4.1.2 원생 조회/필터링**
- 필터: 이름, 학교급, 학년, 학교명, 수업 요일, 성별 (다중 조합)
- 정렬: 이름순/입교일순/학년순
- 재원 상태 필터(재원중/퇴교)

**AC**
- AC-4.1.2-1: 필터 변경 즉시 결과 반영(200ms 이내)
- AC-4.1.2-2: 퇴교 처리된 원생은 기본 미표시, 토글로 표시 가능

**Feature 4.1.3 퇴교 처리**
- 퇴교 처리 시 미사용 보강필요시간이 있는 경우 처리 다이얼로그 표시 (§4.5.6 참조)
- 퇴교 후에도 과거 데이터(출결, 청구 등) 보존 (개인정보 보유 기간 §7 준수)

### 4.2 원생별 수업 정보(수업 스케줄)

**Feature 4.2.1 수업 스케줄 정의**
- 키 구조: **(원생, 요일)** — 한 원생은 한 요일에 한 건만 등록 가능
- 항목: 요일 / 수업 시작 시간 / 1회 수업 시간
- 예: 월·수 13:00 1시간, 금 14:00 2시간 → 주 4시간 / 월 16시간
- 주 총 수업시간 자동 산정 → 표준 교습비 자동 매칭 (§4.3)

**Feature 4.2.2 수업 스케줄 변경(월 단위)**
- 적용 시점: 월 단위(변경월 1일부 적용 또는 사용자 지정 시작일)
- 변경 이력 누적 보존(변경 전·후 시점·내용)

**Feature 4.2.3 선행 수업 (예외 케이스)**

**AC**

### 4.4 학사 스케줄 관리

**Feature 4.4.1 3개월 캘린더 뷰**
- 좌(이전월) / 중앙(기본=다음 달) / 우(이후월) 3개 캘린더 가로 배치
- 중앙 캘린더 년월 좌/우 화살표로 월 이동
- 법정 공휴일·대체휴일 자동 표시

**Feature 4.4.2 교습기간 설정**
- 시작일/종료일 셀 클릭 → "확정" 버튼으로 등록
- 제약
  - 시작일 ≤ 종료일
  - 이전 교습기간과 일자 중첩 불가
  - 월 수업일수 20일 충족을 위해 전·후월 일자 포함 가능
- 확정된 교습기간 셀: 옅은 파스텔 색 배경

**Feature 4.4.3 학사 일정 코드 — 3속성 모델**

각 학사 일정 코드는 다음 3개 속성을 가진다.

| 속성 | 의미 |
|---|---|
| 정규수업 진행 | 해당일에 원생의 정규 수업 출결이 자동 생성되는가 |
| 보강 진행 가능 | 해당일에 보강 출결 등록이 가능한가 |
| 중복불가 | 같은 일자에 동일 코드 중복 등록 차단 |

**Feature 4.4.4 시스템 예약 코드 (5종)**

기본 제공되는 5종 코드는 속성이 잠금되며 사용 여부(활성/비활성)만 조정 가능하다.

| 코드 | 정규수업 진행 | 보강 진행 가능 | 중복불가 | 기간성 | 비고 |
|---|---|---|---|---|---|
| 보강데이 | OFF | ON | ON | 단일 | 정규 비수업일에 보강만 진행 |
| 공휴수업일 | ON | ON | ON | 단일 | 휴일이지만 수업 강행 |
| 방학 | OFF | OFF | OFF | **기간성** | 정규·보강 모두 차단 |
| 단원평가 응시일 | ON | ON | OFF | 기간성(5일) | 정규 수업 + 시험 추가, 보강 허용 |
| 휴원일 | OFF | OFF | ON | 단일 | 단발성 휴원 |

**Feature 4.4.5 사용자 추가 학사 코드**
- 사용자가 신규 코드 추가 가능 (예: 특강일, 학부모 상담일)
- 디폴트 속성값: 정규수업 진행 OFF / 보강 진행 가능 OFF / 중복불가 ON (보수적 디폴트)
- 기간성 여부도 사용자가 선택 가능
- 사용자 추가 코드는 3속성을 자유롭게 편집 가능

**Feature 4.4.6 학사 일정 배치**
- 단일 일자 코드(보강데이/공휴수업일/휴원일): 코드 버튼 선택 → 캘린더 셀 클릭 시 해당일에 등록
- 기간성 코드(방학 등): 시작일·종료일 클릭으로 일괄 등록
- 등록된 일정 셀에 일정명 표시
- 단일 일자 일정은 드래그로 교습기간 내 이동 가능
- 기간성 일정은 시작일·종료일을 다시 지정하여 변경

**Feature 4.4.7 단원평가 응시일 자동 배치**
- 학사 캘린더 생성 시 다음 일정이 자동 배치된다.
  - 1차 단원평가: 교습기간 중 2번째 주 월~금 5일
  - 2차 단원평가: 교습기간 중 4번째 주 월~금 5일
- 원장은 자동 배치된 일정을 드래그 이동, 기간 조정, 추가, 삭제할 수 있다
- 표시 방식: 정규수업 셀과 구분되도록 **셀 상단 띠 배지**로 표기
- 출결 영향: 정규 수업 출결은 그대로 생성됨 (응시는 별도 도메인)

**Feature 4.4.8 보강데이 특수 처리**
- 운영 권장: 4주차 토요일 (확정은 원장 재량)
- 보강데이는 정규 수업이 진행되지 않으며, 보강 필요 원생만 등원한다
- 보강데이에 일괄 보강 등록은 §4.5.4 참조

**Feature 4.4.9 공휴수업일 특수 처리**
- 법정 공휴일·대체휴일·주말이어도 등록 가능
- 등록 시 정규 수업이 진행되는 일자로 산정되어 정규 출결 자동 생성

**AC**
- AC-4.4-1: 지난 달 교습기간/학사일정은 읽기 전용(수정·이동·삭제 모두 차단)
- AC-4.4-2: "정규수업 진행 OFF" 일자에는 정규 출결 자동 생성 없음
- AC-4.4-3: "보강 진행 가능 OFF" 일자에는 보강 등록 시도 시 차단 + 안내
- AC-4.4-4: 중복불가 코드를 같은 일자에 추가 시도 시 경고 후 차단
- AC-4.4-5: 시스템 예약 코드 5종의 속성값은 잠금, 활성/비활성만 토글 가능
- AC-4.4-6: 단원평가 응시일 자동 배치는 학사 캘린더 생성 시 1회 실행되며, 원장이 수정한 후에는 재실행 안 됨

### 4.5 출결/보강 관리

**Feature 4.5.1 월별 출결 생성**
- 월 선택 → "출결생성" 버튼 클릭
- 동작
  1. 해당 월 교습기간 내 각 원생의 수업 요일에 해당하는 일자에 정규 출결 데이터 생성 (디폴트: 출석)
  2. 학사일정 코드의 "정규수업 진행 OFF" 일자는 건너뜀
  3. 원생의 입교일·퇴교일 범위 외 일자는 건너뜀 (재원 기간만 생성)
  4. 소멸기한이 도래한 미보강 결석 데이터는 자동으로 "보강소멸" 상태로 전이
- 중복 생성 방지(이미 생성 시 경고)

**Feature 4.5.2 출결 상태값 정의**

정규 출결의 상태값은 다음 4종으로 구분된다.

| 상태 | 의미 | 보강필요시간 기여 | UI 표시 |
|---|---|---|---|
| 출석 | 정상 출석 | 0 | 검정 |
| 결석 | 결석, 미보강 | 해당 수업시간 | 빨강 |
| 보강완료 | 결석 후 다른 일자에 보강 진행 완료 | 0 | 빨강 + 보강일자 |
| 보강소멸 | 결석 후 소멸기한 도래로 자동 마감 | 0 | 회색 + "보강소멸" |

보강 출결의 상태값은 다음 2종.

| 상태 | 의미 | 비고 |
|---|---|---|
| 보강진행 | 보강 수업 정상 진행 | 충당된 결석을 "보강완료"로 전이 |
| 보강결석 | 보강 약속일에 원생 미등원 | 충당된 결석은 결석 상태 유지 (새 결석 미생성) |

**Feature 4.5.3 출결표 UI 및 상태 토글**
- 행: 재원중 원생 전체
- 컬럼: 일자별 셀 + 요약(출석일수/결석일수/보강필요시간/보강시간)
- 셀 클릭: 출석 ↔ 결석 토글
- 결석 셀: 빨간색 폰트, 결석 사유 메모 입력 가능 (선택)
- 결석으로 변경 → 해당 수업시간만큼 보강필요시간 누적, 소멸기한(년월) 자동 설정
- 결석 → 출석 환원 시 위 효과 역전
- 보강소멸 상태를 결석으로 수동 환원 시 확인 다이얼로그 필수

**Feature 4.5.4 보강 등록 (개별)**
- 원생의 정규 수업이 없는 일자 셀 클릭 (단, 보강필요시간 보유 원생만)
- "보강 진행 가능" 속성이 OFF인 학사일정 일자에는 등록 불가
- 충당할 결석을 다중 선택 (소멸기한 임박 순 정렬)
- 확정 시 보강 출결 1건 생성 + 선택된 결석들이 "보강완료" 상태로 전이
- **매칭 단위는 일(day) 단위** — 결석 1일과 보강 1일이 1:1로 대응
- 보강 1건이 결석 N건 충당 가능 (예: 보강데이 하루로 결석 2일치 처리)

**Feature 4.5.5 보강데이 일괄 등록**
- 진입 경로: 학사 캘린더의 보강데이 셀 클릭 또는 보강 관리 메뉴
- 화면 구성:
  - 보강 필요 원생 리스트 (소멸기한 임박 순 정렬, 색상 강조)
  - 원생별 다중 선택 체크박스
  - 원생별 충당할 결석 일수 지정 (1일 또는 2일 등)
- 일괄 확정으로 선택된 모든 원생의 보강 등록 일괄 처리
- 보강데이 당일 미등원 원생은 사전 취소(보강 등록 삭제) 또는 "보강결석" 처리

**Feature 4.5.6 보강 약속 취소 및 미등원**
- 보강 약속 사전 취소: 해당 보강 레코드 삭제 → 연결된 결석은 자동으로 결석 상태로 환원
- 보강일 당일 원생 미등원: 보강 레코드를 "보강결석" 상태로 마킹 → 충당된 결석은 결석 상태 유지, 새 결석은 생성하지 않음

**Feature 4.5.7 보강 소멸 규칙**
- 소멸기한은 **년월** 단위로 저장 (결석 발생 월 + 1)
- 실제 소멸일은 해당 년월의 교습기간 종료일로 시스템이 동적 판정
- 다음 월 출결 생성 시 소멸기한 도래 + 미보강 결석은 "보강소멸" 상태로 자동 전이
- 다음 월 교습기간 미등록 상태에서는 "소멸기한 미확정"으로 표시, 교습기간 등록 시 확정

**Feature 4.5.8 입교일·퇴교일 변경 시 출결 재조정**
- 출결 생성 후 원생의 입교일 또는 퇴교일을 변경하는 경우, 변경 확인 다이얼로그를 표시한다
- 다이얼로그 표시 정보:
  - 변경 대상 원생명
  - 변경 전·후 입교일(또는 퇴교일)
  - 영향받는 출결 데이터 건수 (삭제될 건수, 신규 생성될 건수)
  - 영향받는 결석·보강 연결 존재 여부 경고
- 원장이 컨펌하면 해당 원생의 출결 데이터를 자동 재조정 (재원 기간 밖 일자 삭제 + 새로 포함된 일자 생성)
- 컨펌하지 않으면 입교일·퇴교일 변경 자체가 취소됨

**Feature 4.5.9 퇴교 시 미사용 보강 처리**
- 퇴교 처리 시 미사용 보강필요시간이 있는 경우 처리 다이얼로그 표시
- 표시 정보: 원생명, 남은 보강필요시간, 미보강 결석 일자 리스트
- 원장 선택지:
  - 즉시 소멸 (전체를 "보강소멸"로 전이)
  - 보강 진행 후 퇴교 (퇴교일을 보류하고 보강 완료 후 다시 퇴교 처리)
  - 외부 처리 후 소멸 (사유 메모 입력, "보강소멸" 전이) — 환불 등 외부 결정 사항
- 환불 금액 자동 계산은 MVP 범위 외

**Feature 4.5.10 결석 이력 조회**
- 원생 상세에서 결석 이력 표 제공
- 컬럼: 결석일 / 수업시간 / 처리 상태 / 보강일 / 소멸기한
- 처리 상태별로 시각적으로 구분 (보강완료/미보강/보강소멸)
- 보강일이 있는 결석은 어떤 보강 일자에 충당되었는지 명확히 표시

**보강필요시간 계산 공식 (단순화)**

```
보강필요시간 = Σ(정규 출결.수업시간 WHERE 상태 = 결석)
```

"보강완료"와 "보강소멸"은 정산 종료 상태이므로 합산에서 제외된다.

**AC**
- AC-4.5-1: 출결 생성은 동일 월에 1회만 가능, 재실행은 v2.0 이후 검토
- AC-4.5-2: 결석↔출석 토글 시 보강필요시간이 정확히 ±변경
- AC-4.5-3: 보강은 일 단위로 결석에 매핑되며 1:N 매칭 가능 (보강 1건이 결석 N건 충당)
- AC-4.5-4: 보강 삭제 후 데이터 정합성 유지 (연결된 결석들이 결석 상태로 환원)
- AC-4.5-5: 보강소멸 상태를 결석으로 수동 환원할 때 확인 다이얼로그 필수
- AC-4.5-6: 입교일·퇴교일 변경 시 컨펌 다이얼로그 후에만 출결 재조정 실행
- AC-4.5-7: 결석 이력에서 "보강완료"와 "보강소멸"이 명확히 시각적으로 구분되어야 함

### 4.6 수업 관리(캘린더 뷰)

**Feature 4.6.1 일/주/월 뷰(Outlook 스타일)**
- 날짜 셀 상단: 시간대별 총 수업 인원수
- 시간대 구분선(1시간 단위) 아래 원생 이름 + 당일 수업 시간 (한 줄 3명씩)

**Feature 4.6.2 원생 상세 팝업(출결·보강 상세)**
- 표시 항목: 이름, 학년, 정규/보강 구분, 시작·종료 시간, 해당 월 결석일·미수업 시간·소멸기한, 보강여부·보강일·잔여 보강필요시간
- "출결/보강관리" 버튼: 해당 원생으로 필터링된 출결관리 화면으로 이동

**Feature 4.6.3 보강 관리 전용 뷰**
- 보강 필요 원생 리스트 (이름, 결석일, 미수업 시간, 소멸기한, 보강 여부, 보강일, 잔여 보강필요시간)
- 정렬: 소멸기한 임박 순
- 소멸 임박(소멸기한이 해당 월의 교습기간 종료일에서 7일 이내) 행 강조 표시
- 보강데이 일괄 등록 진입 버튼 제공

**AC**
- AC-4.6-1: 시간대별 인원수 = 해당 시간대에 수업 시작하는 원생 + 진행 중 원생 합산
- AC-4.6-2: 보강 관리 뷰에서 소멸 임박 데이터를 한눈에 식별 가능 (색상/아이콘)

### 4.7 단원평가 점수 입력

**Feature 4.7.1 단원평가 회차 등록**
- 학사 캘린더의 단원평가 응시일 자동 배치(§4.4.7)와 별개로, 단원평가 회차(이벤트)는 점수 입력 메뉴에서 관리한다
- 회차 정보: 회차명(예: 2026-06 1차), 응시 기간(시작일/종료일, 자동 배치값을 디폴트로 사용), 학년별 평가대상 단원 설명
- 월 2회 기본(2주차/4주차), 원장이 자유롭게 추가·수정·삭제 가능

**Feature 4.7.2 원생별 점수 입력**
- 원생별 점수는 2회 입력
  - 1차: 단원평가 채점 결과
  - 2차: 오답 재시험 반영 결과
- 응시일(`test_date`): 원생이 실제로 시험을 치른 일자 (응시 기간 외 일자도 입력 가능, "기간 외 응시" 표시)
- 미응시 원생은 "미응시" 상태로 표시 + 알림

**Feature 4.7.3 학습 성과 추이 조회**
- 학년별 학습 성과 추이
- 원생별 학습 성과 추이 (라인 차트)

**AC**
- AC-4.7-1: 점수 범위 0~100, 정수만 허용
- AC-4.7-2: 1차 점수 미입력 시 2차 입력 차단
- AC-4.7-3: 응시 기간 외 응시는 허용하되 "기간 외 응시"로 시각적 표시
- AC-4.7-4: 미응시 원생은 단원평가 메뉴에서 명확히 식별 가능

### 4.8 단원평가 및 학습보고서 작성

학습보고서는 단원평가 점수에 종속된 도메인으로, **분기 단위**로 작성된다. 분기 6회 단원평가(분기 3개월 × 월 2회) 점수를 종합하여 학부모에게 학습 진척과 종합 의견을 전달한다.

**Feature 4.8.1 보고서 메뉴 진입**
- 분기 선택 (디폴트: 가장 최근 완료된 분기)
- 분기 정의: 학사력 기준 — 3~5월 / 6~8월 / 9~11월 / 12~2월
- 분기 마지막 월의 2차 단원평가 점수까지 모두 입력 완료된 분기만 보고서 작성 가능
- 분기 내 단원평가 대상 원생 리스트 표시
- 원생 선택 시 보고서 입력 화면 진입

**Feature 4.8.2 보고서 입력 화면**

화면은 참고 정보 영역(상단, 읽기 전용)과 입력 영역(하단)으로 구성된다.

**(가) 참고 정보 영역**
- 원생명, 학년, 학교
- **해당 분기 단원평가 6회 점수** (회차별 1차·2차, 미시행 회차는 "—" 표기 없이 실제 시행 회차만 표시)
- **당해 연도 모든 과거 분기 점수 내역** (분기별 평균·회차별 점수 표)
- **점수 추이 차트**: 가로축 회차(전체 분기 통합), 1차·2차 점수를 회차마다 점으로 표시 + 분기별 평균 라인 중첩

**(나) 입력 영역**
- **종합의견** (멀티라인 입력, 줄바꿈 허용)

**Feature 4.8.3 보고서 저장**
- 저장 키: **(분기, 원생)** 단위로 1건
- 후속 수정·재인쇄 가능
- 단원평가 점수 데이터는 직접 참조 (저장 시 복사 보관하지 않음 — 점수 수정 시 보고서 표시도 자동 반영)

**Feature 4.8.4 보고서 출력**
- 출력 방식: **인쇄 직접 출력 또는 PDF 저장** (§4.13.2 분기별 일괄 내보내기 참조)
- 제목: "수학단원평가 및 학습보고서 — [분기 표기]"
- 레이아웃: A4 1장에 4명, 4등분 박스 형태
- 박스 내부 수직 배치:
  1. **상단**: 원생 성명 + 분기 단원평가 점수표 (회차 × 1차·2차)
  2. **중단**: 점수 추이 그래프 (회차별 점수 + 분기 평균 라인)
  3. **하단**: 종합의견 (멀티라인, 줄바꿈 반영)
- 디자인 원칙: 심플하고 가독성 우선, 박스 비율 균등

**AC**
- AC-4.8-1: 보고서 입력 화면에서 해당 분기 6회 점수 + 당해 연도 모든 과거 분기 점수 조회 가능
- AC-4.8-2: 점수 추이 차트는 회차별 1차·2차 점수를 점으로 모두 표시하고 분기별 평균 라인을 중첩 표시
- AC-4.8-3: 멀티라인 입력 내용의 줄바꿈은 인쇄 시 그대로 반영
- AC-4.8-4: A4 1장 4분할 인쇄 시 박스 비율 균등 유지
- AC-4.8-5: 보고서는 단원평가 점수에 종속되어 저장 — 점수 수정 시 보고서 표시도 자동 반영
- AC-4.8-6: 분기 마지막 월의 2차 단원평가 점수 미입력 상태에서는 보고서 작성 차단 (안내 메시지 표시)
- AC-4.8-7: 분기 단원평가가 6회 미만 시행된 경우 실제 시행 회차만 표시 (점수표·차트 동적 구성)

### 4.9 교습비 청구/수납 관리

**Feature 4.9.1 청구 데이터 생성**
- 년월 선택(디폴트: 마지막 교습기간 월)
- "청구 데이터 생성" 버튼 → 재원 원생 대상으로 일괄 생성
- 청구 항목: 성명, 청구년월, 주단위 총 수업시간, 청구 교습비, 확정 여부, 월 중 입퇴교 플래그
- 청구 교습비 = 표준 교습비 매핑값(주 수업시간 기준)
- 동일 월 중복 생성 시 경고 후 차단

**Feature 4.9.2 월 중 입퇴교 처리**
- 월 중 입교·퇴교 원생도 표준 청구액으로 일괄 생성
- 월 중 입퇴교 원생은 청구 화면에 시각적으로 구분 표시 (입교일·퇴교일 라벨, 배경색 등)
- 원장이 해당 원생의 청구액을 수동 조정 후 확정 처리
- 일할 자동 계산은 MVP 범위 외 (Post-MVP)

**Feature 4.9.3 청구 확정 상태 관리**
- 청구 데이터는 "미확정" 상태로 생성됨
- 원장이 검토 완료 후 "확정" 처리
- 확정된 청구는 수정 시 사용자 확인 단계 필수
- 미확정 청구는 청구 화면 진입 시 상단 배너로 건수 표시

**Feature 4.9.4 청구 화면 정렬**
- 디폴트 정렬: 미확정 + 월 중 입퇴교 원생을 상단에 우선 배치
- 원장이 우선 처리 대상을 즉시 인지할 수 있도록 시각화

**Feature 4.9.5 수납 관리**
- 수납 항목: 성명, 청구월, 입금여부, 입금일, 입금자명, 결제수단(코드), 매입사명(코드, 카드결제 시)
- 결제수단 코드: 계좌이체, 현금, 신용카드, 체크카드, 결제선생, 지역화폐 (사용자 추가 가능)
- 카드사 코드: 삼성, 신한, KB국민, 우리, 현대, 롯데, 카카오뱅크, 케이뱅크 (사용자 추가 가능)

**Feature 4.9.6 입금 일괄 처리 모드**
- 청구 화면에 "입금 일괄 처리" 진입 버튼 제공
- 미입금 청구 리스트 한 화면에 표시
- 행별 빠른 입력: 체크박스 + 입금일 + 입금자명 + 결제수단 단일 행에서 입력
- 일괄 확정으로 다수 입금을 한 번에 처리
- 미납 원생 자동 추출 → 별도 안내 리스트로 활용 (안내 발송은 원장 재량, 시스템 자동 발송 없음)

**Feature 4.9.7 월별 청구 마감 (Closing)**
- 청구 데이터는 다음 3단계 상태로 관리된다.

| 상태 | 의미 | 수정 가능성 |
|---|---|---|
| 미확정 | 원장 검토 대기 (월 중 입퇴교 등) | 자유 |
| 확정 | 원장이 검토 완료한 청구 | 확인 다이얼로그 후 가능 |
| 마감 | 월별 청구 전체 처리 완료 후 잠금 | 추가 경고 + 강한 확인 후에만 가능 |

- 마감 액션: 청구 화면에 "당월 청구 마감" 버튼 (해당 월의 모든 청구가 "확정" 상태일 때만 활성화)
- 마감 후 효과
  - 청구 데이터 수정 시 강한 경고 + 원장이 사유 입력 필요
  - 대시보드에 "당월 청구 마감 완료" 표시
  - 미납률 KPI 집계 기준일 확정

**AC**
- AC-4.9-1: 청구액 = (주 수업시간 → 표준 교습비 매핑) 조정 전 기본값
- AC-4.9-2: 월 중 입퇴교 원생도 표준 청구액으로 생성되되 화면에서 시각 구분
- AC-4.9-3: 청구 확정 후 금액 수정 시 사용자 확인 다이얼로그 필수
- AC-4.9-4: 결제수단이 카드 계열일 때만 매입사명(카드사) 필수 입력
- AC-4.9-5: 미확정 청구가 존재할 때 청구 화면 진입 시 상단 배너로 건수 표시
- AC-4.9-6: 입금 일괄 처리 모드에서 한 화면에 최소 20행 이상 표시
- AC-4.9-7: 청구 마감은 해당 월 모든 청구가 "확정" 상태일 때만 가능
- AC-4.9-8: 마감 후 청구 데이터 수정 시 사유 입력 필수

### 4.11 대시보드

**Feature 4.11.1 교습소 현황 위젯**
- 현재 재원 총원
- 분기별 신규 입교/퇴교 인원
- 학년별 / 성별 / 학교별 인원수·비율

**Feature 4.11.2 당일 수업 정보**
- 시간대별 수업 인원 및 명단

**Feature 4.11.3 해당 월 핵심 요약**
- 출결 입력 진행률
- 총 청구액 / 입금 완료액 / 미납액
- 당월 신규 입교/퇴교 원생 수

**Feature 4.11.4 업무 리마인더 및 알림 (강화)**

대시보드 상단에 운영상 즉시 인지가 필요한 알림을 모아 표시한다. 누락 방지가 KPI 직접 달성에 기여한다.

| 알림 종류 | 트리거 조건 | 우선순위 표시 |
|---|---|---|
| 출결 미입력 알림 | 오늘 또는 이번 주 내 미입력 수업 존재 | 빨강 (즉시 처리 권장) |
| 보강 소멸 임박 알림 | 소멸기한이 해당 월 교습기간 종료일 기준 7일 이내인 보강필요시간 보유 원생 존재 | 주황 |
| 미확정 청구 알림 | 청구 데이터 중 "미확정" 상태 건수 | 주황 |
| 다음 달 학사 미수립 경고 | 현재 월 25일 이후 진입 시 다음 달 교습기간 미등록 상태 | 주황 (D-7) → 빨강 (D-3) |
| 운영 일정 알림 | 이번 주 보강데이 D-3, 단원평가 응시 D-5 등 | 파랑 (정보성) |

- 알림 클릭 시 해당 처리 화면으로 직접 이동
- 알림은 처리 완료 시 자동 해제

**Feature 4.11.5 출결 입력 진행률 상세**
- §4.11.3의 출결 입력 진행률 위젯 확장
- 미입력 일자 클릭 시 해당 일자의 출결관리 화면으로 즉시 이동

### 5.1 Platform & Technology Stack

  - Tauri 권장 사유: 메모리 효율, 작은 번들 크기, Rust 기반 보안 모델
  - Electron 대체 사유: 생태계 성숙도, 개발 속도 우위

- **데이터 저장(단일 소스 오브 트루스)**: **로컬 SQLite + SQLCipher(AES-256 암호화)**
- **데이터 공유**: OS 레벨 **클라우드 동기화 폴더**(네이버 MYBOX 우선, 미지원 환경 대비 iCloud Drive / Dropbox 대체)
- 캘린더 뷰: FullCalendar 또는 React Big Calendar
- 이미지 생성: HTML5 Canvas + html-to-image

**비용**
- 월 0원 (네이버 MYBOX 무료 30GB 한도 내 운영)

### 5.3 데이터 동기화 전략 (로컬 + 클라우드 폴더)

**기본 모델**
- DB 단일 파일(`app.db`)을 양 PC가 공유하는 클라우드 동기화 폴더에 둠
- 양 PC는 시점 분리 사용을 원칙(동시 사용 비권장)
- 모든 동기화는 OS 클라우드 클라이언트(MYBOX 등)에 위임

**동시성 제어 (Lock 메커니즘)**
- 앱 시작 시 동기화 폴더의 `app.lock` 파일 확인
  - 락 없음 → 락 생성(디바이스 ID + 타임스탬프) 후 진입
  - 락 존재 + 다른 디바이스 점유 중 → 경고 화면 표시 후 다음 옵션 제공:
    - "다른 PC에서 종료 후 다시 시도" (기본)
    - "강제 점유" (5분 이상 갱신 없는 락만 가능 — 비정상 종료 케이스 복구용)
- 앱 사용 중 매 60초마다 락 타임스탬프 갱신(heartbeat)
- 정상 종료 시 락 자동 해제

**동기화 완료 대기**
- 앱 시작 직후 DB 파일·락 파일의 최신 동기화 여부 확인
- 미완료 감지 시 "동기화 대기 중" 화면 표시, 완료 후 자동 진입
- 30초 경과 시 사용자에게 수동 새로고침 옵션 제공

**무결성 검증**
- 앱 시작 시 `PRAGMA integrity_check` 자동 실행
- 손상 감지 시 가장 최근 정상 백업으로 자동 복원 + 사용자 알림 + 손상본 격리 보관

### 5.4 자동 백업 정책 (4계층)

| 계층 | 트리거 | 보관 개수 | 저장 경로 |
|---|---|---|---|
| 종료 백업 | 앱 정상 종료 시 | 최근 10개 | `backup/exit/` |
| 시간 백업 | 사용 중 1시간마다 | 최근 24개 | `backup/hourly/` |
| 일일 백업 | 매일 자정 직후 첫 실행 시 | 30일치 | `backup/daily/` |
| 주간 백업 | 매주 월요일 첫 실행 시 | **4주치** | `backup/weekly/` |

**기술 구현**
- SQLite Online Backup API 사용 → 트랜잭션 중에도 안전한 스냅샷 생성
- 파일명: `app_YYYYMMDD_HHMMSS.db` (자동 순환 삭제)
- 모든 백업은 SQLCipher 암호화 상태 유지
- 사용자 수동 백업: "지금 백업" 버튼 + 사용자 지정 경로로 단발성 내보내기

**복원 절차**
- 설정 메뉴 → 백업 관리 → 백업 목록(시점·크기·계층 표시) → 미리보기 → 복원
- 복원 직전 현재 DB를 자동으로 `restore_rollback/`에 보존 (실수 대비)

**복원 리허설 모드 (필요시 수행)**
- 진입: 설정 메뉴 → 백업 관리 → "백업 복원 리허설" 버튼
- 사용 시점: 백업 정상 작동 검증 필요 시, DB 손상 의심 신호(이상 오류·속도 저하) 발생 시, 중요 작업 전 안전망 확인 시
- 동작
  1. 사용자가 백업 목록 중 하나 선택 (디폴트: 가장 최근 백업)
  2. 시스템이 별도 임시 환경에 백업 파일 복원 시도
  3. 무결성 검증(`PRAGMA integrity_check`) 자동 실행
  4. 결과 표시 (성공/실패, 검증된 데이터 건수, 손상 항목 등)
  5. 임시 환경 자동 삭제
- 실행 결과는 운영 데이터에 영향 없음 (격리 환경)
- 정기 알림 없음 — 사용자가 필요할 때만 1클릭으로 실행

### 5.5 인증·접근 제어
- 앱 시작 시 비밀번호(또는 OS 생체인증, Touch ID/Windows Hello) 잠금
- DB 암호화 키: 사용자 비밀번호 기반 PBKDF2 유도 + OS Keychain/Credential Manager 보관
- 외부 네트워크 통신 없음(클라우드 동기화는 OS 클라이언트가 담당, 앱은 로컬 파일만 접근)

### 5.6 성능 요구사항
- 화면 전환 < 300ms
- 출결표 50명 × 31일 렌더링 < 1초
- 청구 데이터 50명 생성 < 3초
- 공지문 이미지 50장 일괄 생성 < 30초
- 앱 시작 ~ 메인 화면 진입(락 확인 + 무결성 검증 포함) < 3초

### 5.7 접근성·UX 표준(50대 사용자 친화)

**시각·조작 접근성**
- 기본 폰트: Pretendard 18pt(헤더 24pt+), 행간 1.5
- 색상: 저자극 톤(차분한 베이지/연그레이 배경, 강조색은 무채도 기반)
- 명도 대비 ≥ 4.5:1(WCAG AA)
- 클릭 가능 영역 최소 44×44px
- 키보드 단축키 + 마우스 조작 병행 지원
- 상단 상태바: 현재 점유 디바이스, 마지막 백업 시각, 동기화 상태 항시 표시

**키보드 단축키 체계**

50대 사용자는 마우스 위주 사용이 기본이며, 단축키는 익숙해진 후 효율을 높이는 보조 수단으로 정의한다.

| 단축키 | 동작 | 적용 범위 |
|---|---|---|
| F1 | 도움말 | 전체 화면 공통 |
| Ctrl+F | 글로벌 검색바 포커스 | 전체 화면 공통 |
| Ctrl+N | 신규 원생 등록 | 원생 관리 화면 |
| Ctrl+S | 현재 양식 저장 | 입력 화면 |
| Ctrl+Z | 직전 작업 Undo | 출결·청구 조정 등 주요 화면 |
| ESC | 다이얼로그/팝업 닫기, 검색바 닫기 | 전체 화면 공통 |
| Ctrl+P | 인쇄 (학습보고서 등) | 출력 가능 화면 |

- 단축키는 메뉴에 단축키 표기 병기 (예: "신규 원생 등록 (Ctrl+N)")
- 도움말(F1)에서 전체 단축키 목록 조회 가능

**실수 복구 메커니즘**
- 양식 입력 화면(원생 등록·수정, 학습보고서 작성, 청구 조정 등)은 작성 중 자동 임시저장 (3분 단위)
- 출결 토글·청구 금액 조정·보강 등록 등 주요 행위는 직전 1단계 Undo 지원
- 메뉴 이동 또는 창 닫기 시 미저장 변경분이 있으면 경고 다이얼로그 표시
- 모든 위험 동작(삭제, 청구 재생성, 보강 삭제, 백업 복원, 보강소멸 환원)에 명시적 확인 대화상자
- 임시저장 데이터는 다음 진입 시 자동 복원 + 사용자에게 "이전 작업 이어하기" 선택지 제공

## 6. Data & Quality Requirements

### 6.1 핵심 도메인 개념도

데이터 영역은 다음 도메인 그룹으로 구성된다. (개념 수준이며 구체 테이블·필드 설계는 개발 단계에서 결정)

```
[원생 관리 도메인]
  - 원생 (기본 정보 + 성별)
  - 원생별 수업 스케줄 (원생 × 요일, 한 요일에 1건)
  - 학교 (코드)

[학사 도메인]
  - 교습기간 (월별)
  - 학사 일정 (일자 × 코드)
  - 학사 일정 코드 (시스템 예약 5종 + 사용자 추가, 3속성: 정규수업 진행 / 보강 진행 가능 / 중복불가)

[출결·보강 도메인]
  - 정규 출결 (원생 × 일자, 상태: 출석/결석/보강완료/보강소멸)
  - 보강 출결 (원생 × 일자, 상태: 보강진행/보강결석)
  - 보강-결석 매칭 (일 단위 1:N, 보강 1건이 결석 N건 충당 가능)

[단원평가 도메인]
  - 단원평가 회차 (응시 기간, 학년별 단원 정보)
  - 원생별 점수 (1차/2차, 응시일)
  - 분기 학습보고서 (분기 × 원생, 종합의견)
    └ 분기 6회 단원평가 점수에 종속 (점수 데이터를 직접 참조)

[청구·수납 도메인]
  - 표준 교습비 (코드: 주 수업시간 ↔ 금액)
  - 청구 (원생 × 청구년월, 확정 상태, 월 중 입퇴교 플래그)
  - 수납 (청구당 1건)
  - 결제수단 / 카드사 (코드)
```

### 6.2 핵심 비즈니스 키 & 제약

- 원생: 일련번호(법정번호) UNIQUE
- 수업 스케줄: (원생, 요일) UNIQUE — 한 원생은 한 요일에 한 건만
- 정규 출결: (원생, 일자) UNIQUE
- 보강 출결: 같은 일자 중복 허용 (한 일자에 여러 원생 보강 가능)
- 분기 학습보고서: (분기, 원생) UNIQUE
- 청구: (원생, 청구년월) UNIQUE
- 학사 일정: 중복불가 코드는 (일자, 코드) UNIQUE
- 교습기간: 일자 중첩 금지

### 6.3 검증 규칙
- 입력 시 클라이언트(앱) 즉시 검증 + DB 제약(트리거/CHECK) 이중 검증
- 전화번호: 010-XXXX-XXXX 형식 권장(자유 입력 허용, 형식 불일치 시 경고)
- 일자: 미래 일자에 출결 입력 시 경고
- 청구액: 0 이상 정수
- 단원평가 점수: 0~100 정수

### 6.4 에러 처리·복구
- 사용자 친화 메시지(기술 에러 코드 비노출)
- DB 손상 감지: 가장 최근 정상 백업으로 자동 복원 + 사용자 알림
- 동기화 폴더 접근 실패: 읽기 전용 모드로 전환 + 마지막 정상 상태 안내
- 락 충돌 감지: 다른 디바이스 점유 안내 + 5분 미갱신 락 강제 해제 옵션
- 모든 쓰기 작업: 트랜잭션 + audit log 보존
- 치명적 오류: 자동 크래시 리포트(개인정보 제외) 옵트인

### 6.5 테스트 전략
- 단위 테스트: 비즈니스 규칙(보강 일 단위 매칭, 소멸 처리, 청구 계산) 100% 커버
- 통합 테스트: 출결 생성 → 결석 → 보강 등록 → 보강 삭제 시나리오
- E2E: 핵심 사용자 흐름(UC-1 ~ UC-6) Tauri WebDriver(`tauri-driver`) 기반 — Tauri 데스크톱 셸을 직접 제어하는 공식 권장 방식. 일반 Playwright(브라우저 전용)는 Tauri WebView 제어 불가
- 사용자 수용 테스트(UAT): 원장 2주 파일럿(실데이터 일부 마이그레이션)
- **복원 리허설 품질 기준**: 복원 리허설 실행 시 무결성 검증 100% 통과

### 6.6 데이터 자가 진단

**Feature 6.6.1 월별 자가 진단 리포트**

데이터 무결성 오류를 사후에 누적된 상태로 발견하지 않도록, 월 1회 자동 검사 후 이상 항목을 대시보드로 안내한다.

**검사 항목 (예시)**
- 보강필요시간 음수 또는 이상값 보유 원생
- 재원중 원생이지만 당월 출결 데이터 미생성
- 재원중 원생이지만 당월 청구 데이터 미생성
- 수업 스케줄 vs 출결 데이터 불일치 (스케줄 변경 후 미반영)
- 결석 데이터에 소멸기한 미설정
- 보강 출결이 어떤 결석에도 연결되지 않은 고아 데이터
- 청구·수납 데이터 정합성 (입금 완료 표시 + 입금 정보 누락 등)

**Feature 6.6.2 진단 실행 방식**
- 자동: 매월 1일 첫 실행 시 자동 수행
- 수동: 설정 메뉴 → "데이터 자가 진단" 버튼으로 즉시 실행 가능
- 결과 보관: 진단 이력 누적 (최근 12개월)

**Feature 6.6.3 진단 결과 대응**
- 이상 항목 발견 시 대시보드 알림 위젯에 표시 (주황 우선순위)
- 알림 클릭 시 진단 상세 화면으로 이동 → 항목별 해결 가이드 표시
- 해결 후 재진단 가능

**AC**
- AC-6.6-1: 매월 1일 첫 실행 시 자가 진단 자동 수행
- AC-6.6-2: 진단 결과 이상 항목 0건이면 대시보드 알림 미표시
- AC-6.6-3: 이상 항목 발견 시 1클릭으로 해당 항목의 처리 화면으로 이동 가능
- AC-6.6-4: 진단 이력은 최근 12개월 보관

## 7. Security & Compliance

### 7.1 개인정보보호법 준수
- 수집 항목: 학생 성명, 학교, 학년, 전화번호(학생/부모) → 최소수집 원칙
- 처리 목적: 교습 서비스 제공 및 청구
- 보유 기간: 퇴교 후 3년(학원법/교습소 운영 기준)
- 원장은 별도 개인정보 수집·이용 동의서를 학부모로부터 서면 확보(앱 외부 절차)

### 7.2 데이터 보호
- 로컬 DB(`app.db`): AES-256(SQLCipher), 사용자 비밀번호 기반 키 유도
- 백업 파일: SQLCipher 암호화 그대로 유지
- 클라우드 동기화 폴더(MYBOX 등): OS 클라이언트의 자체 TLS 전송 보안에 위임
- 암호화 키: OS Keychain(macOS) / Credential Manager(Windows) 보관
- 사용자 비밀번호 분실 대비: 복구 코드(12자리) 발급 후 안전한 곳 보관 안내

### 7.3 감사 로그
- 모든 쓰기 작업: 일시, 변경 전/후, 사용자 ID
- 보관: 1년 롤링
- 우선 기록: 출결 변경, 청구 변경, 코드 변경, 원생 등록·퇴교 처리

### 7.4 비고
- 의료 도메인이 아니므로 ISMS-P / 의료법은 비적용
- 카카오톡 자동 발송은 MVP 범위 외(이미지 저장 후 수동 발송)

## 8. Constraints & Assumptions

### 8.1 기술 제약
- 단일 사용자(원장) 가정, 멀티 사용자 시나리오 미고려
- 카카오톡 비즈니스 API는 사업자 인증 등 별도 요건 → MVP 제외
- 양 PC 동시 사용 비권장 → 앱 레벨 **락 파일(`app.lock`)** 로 직렬화
- 데이터 동기화는 OS 클라우드 클라이언트(MYBOX 등)에 위임 → 동기화 지연·실패 시 앱이 감지·안내

### 8.2 비즈니스 제약
- 원장 1인 운영, 사용 시간 한정(수업 외)
- 일할 계산 자동화는 v2.0
- 회계·세무 기능은 Post-MVP

### 8.3 가정
- 양 PC는 시점 분리 사용(예: 교습소 종료 → 동기화 완료 확인 → 자택 사용)
- 클라우드 동기화 폴더 정상 작동(MYBOX 클라이언트 양 OS 설치·로그인)
- 원장은 카카오톡 PC/모바일 사용 가능(생성된 이미지 수동 발송)
- 월 평균 원생 50명 이내(성능 기준)
- 인터넷 단절 중에도 로컬 작업 가능(연결 복구 시 OS가 자동 동기화)

## 9. Success Metrics (KPI)

| KPI | 측정 방식 | 목표 |
|---|---|---|
| 교습비 미납률 | 미입금 청구건 / 전체 청구건 | ≤ 5% |
| 출결 입력 완료율 | 입력 일수 / 교습기간 일수 × 100 | ≥ 95% |
| 월 행정 처리 시간 | 원장 자가 보고(월말 설문) | ≤ 4시간 |
| 사용자 만족도 | 분기별 자가 평가(5점 척도) | ≥ 4.5 |
| 데이터 정합성 오류 | 월별 자가 진단(§6.6) 이상 항목 건수 | = 0 |
| 백업 무결성 검증 통과율 | 일일 자동 `PRAGMA integrity_check` 통과 비율 | = 100% |
| 시스템 가용성 | 월간 정상 동작 시간 비율 | ≥ 99% |
| 청구 → 입금 평균 소요 일수 | (입금일 − 청구일) 평균 | ≤ 7일 |

> 참고: "백업 복원 성공률"은 정기 KPI에서 제외되었다. 복원 리허설은 필요시 수행 모드로 운영되며, 실행 시 무결성 100% 통과는 §6.5 테스트 전략의 품질 기준으로 관리한다.

## 10. Appendix

### 10.1 Glossary
- **교습기간**: 월별 수업 시작일 ~ 종료일 (월 20일 충족, 전·후월 일자 포함 가능)
- **분기**: 학사력 기준 3개월 단위. 3~5월(1Q) / 6~8월(2Q) / 9~11월(3Q) / 12~2월(4Q)
- **분기 단원평가 6회**: 분기 3개월 × 월 2회 단원평가 = 분기당 총 6회 시행 기준 (실제 시행 횟수는 운영 상황에 따라 가변)
- **분기 학습보고서**: 분기 단위로 작성되는 학부모용 학습 평가서. 분기 6회 단원평가 점수에 종속
- **보강필요시간**: 결석으로 발생한 미수업 시간 누적값 (시간 단위)
- **소멸기한**: 미보강 결석이 효력을 잃는 기한. **년월** 단위로 저장(결석 발생 월 + 1), 실제 소멸일은 해당 년월의 교습기간 종료일
- **보강데이**: 교습소 공통 보강일 (통상 4주차 토요일). 정규 수업은 없고 보강 필요 원생만 등원
- **공휴수업일**: 법정 공휴일·주말이지만 수업을 진행하는 날
- **보강완료**: 결석에 대해 실제 보강 수업이 진행되어 정산이 종료된 상태
- **보강소멸**: 결석에 대한 보강이 소멸기한 내에 진행되지 않아 자동 마감된 상태
- **표준 교습비**: 주 단위 수업시간에 매핑된 기본 청구액
- **재원생**: 입교일 ≤ 오늘 ≤ 퇴교일(없으면 ∞)인 원생
- **시스템 예약 코드**: 학사 일정 코드 중 시스템 기본 제공 5종(보강데이/공휴수업일/방학/단원평가 응시일/휴원일), 속성 잠금
- **확정/미확정 청구**: 청구 데이터의 검토 상태. 미확정은 원장 검토 대기 (특히 월 중 입퇴교 건)

### 10.2 비즈니스 규칙 요약(핵심)

**(1) 출결 생성 규칙**
- 교습기간 내, 원생의 수업 요일 일자에만 정규 출결 생성
- 학사 일정 코드의 "정규수업 진행 OFF" 일자는 생성 제외
- 원생 입교일·퇴교일 범위 외 일자는 생성 제외 (재원 기간만 생성)
- 디폴트 상태: 출석
- 다음 월 출결 생성 시 소멸기한 도래 미보강 결석은 "보강소멸" 자동 전이

**(2) 출결 상태값**
- 정규 출결: 출석 / 결석 / 보강완료 / 보강소멸
- 보강 출결: 보강진행 / 보강결석

**(3) 보강 매칭 규칙**
- 매칭 단위: 일(day) 단위. 결석 1건 ↔ 보강 1건 (시간 단위 부분 보강 없음)
- 보강 1건이 여러 결석 충당 가능 (보강데이 하루로 결석 2일치 처리 등)
- 보강 등록 시 충당할 결석을 다중 선택 (소멸기한 임박 순 권장)
- 보강 약속 사전 취소 = 보강 레코드 삭제 → 연결된 결석은 결석 상태로 환원
- 보강일 당일 미등원 = "보강결석" 마킹, 결석은 결석 상태 유지, 새 결석 미생성

**(4) 보강필요시간 계산**
- 보강필요시간 = Σ(정규 출결 수업시간 WHERE 상태 = 결석)
- 보강완료·보강소멸은 합산 제외

**(5) 소멸 처리**
- 소멸기한 = 결석 발생 월 + 1 (년월 단위 저장)
- 실제 소멸일 = 해당 년월의 교습기간 종료일 (시스템 동적 판정)
- 다음 월 교습기간 미등록 상태에서는 "소멸기한 미확정" 표시
- 보강소멸 → 결석 수동 환원은 확인 다이얼로그 필요 (예외 처리용)

**(6) 청구 규칙**
- 청구액 기본값 = 표준 교습비 매핑(주 수업시간)
- 월 중복 청구 불가
- 월 중 입퇴교는 표준 청구액으로 일괄 생성 + 화면 시각 구분 + 원장 수동 조정 후 확정
- 청구는 미확정/확정 두 상태 관리, 미확정 건수는 대시보드와 청구 화면 상단에 알림

**(7) 퇴교 시 미사용 보강 처리**
- 퇴교 처리 시 보강필요시간 보유 원생에 대해 처리 다이얼로그 표시
- 선택지: 즉시 소멸 / 보강 진행 후 퇴교 / 외부 처리 후 소멸(메모 입력)

**(8) 입교일·퇴교일 변경 시 출결 재조정**
- 변경 영향(삭제·신규 건수, 보강 연결 경고)을 다이얼로그로 표시
- 원장 컨펌 시에만 출결 데이터 자동 재조정 실행
- 컨펌하지 않으면 입교일·퇴교일 변경 자체가 취소됨

**(9) 학사 일정 코드 속성**
- 3속성: 정규수업 진행 / 보강 진행 가능 / 중복불가
- 시스템 예약 5종은 속성 잠금, 사용 여부만 토글
- 사용자 추가 코드는 3속성 자유 편집, 디폴트 모두 보수적(정규수업 OFF / 보강 OFF / 중복불가 ON)

**(10) 단원평가 응시일 자동 배치**
- 학사 캘린더 생성 시 1차(2주차 월~금) + 2차(4주차 월~금) 자동 배치
- 원장이 자유롭게 수정·이동·삭제 가능

**(11) 공지문 파일 경로 규칙**
- `[설정 폴더]/[YYYYMM]/[YYYYMM]_[원생이름].png`

**(12) 학습보고서 출력 규칙**
- 작성 단위: 분기 (학사력 기준, 3·6·9·12월 시작)
- 작성 가능 시점: 분기 마지막 월의 2차 단원평가 점수 입력 완료 후
- 인쇄 직접 출력을 기본으로 하되 PDF 저장도 허용 (분기별 PDF 일괄 내보내기는 §4.13 참조)
- A4 1장에 4명, 4등분 박스 형태
- 박스 내부 수직 배치: 상단(원생 성명 + 분기 점수표) / 중단(추이 그래프) / 하단(종합의견)
- 단원평가 6회 미만 시행 시 실제 시행 회차만 동적 표시
